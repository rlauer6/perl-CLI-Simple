# CLI::Simple 2.0.0 Release Notes

## Overview

Version 2.0.0 is a major release introducing optional role-based
architecture for larger CLI applications, a suite of scaffolding tools
for new and migrating projects, and bash completion
generation. Backward compatibility with existing single-module scripts
is fully preserved.

## New Features

**Role-based architecture via YAML manifest.** Scripts can now declare
commands and options in a YAML manifest and implement each command in
a dedicated `Role::Tiny` role. `CLI::Simple` handles role composition
and dispatch table construction automatically at compile time:

```perl
use CLI::Simple qw(:roles);
use parent qw(CLI::Simple);

caller or exit __PACKAGE__->main;
```

The manifest filename is derived by convention from the module name -
`My::Script` looks for `my-script.yml`.

**`%INTERNAL_COMMANDS` dispatch pattern.** Internal commands prefixed
with `-` are intercepted before `GetOptions` runs, enabling
framework-level operations that don't require a fully constructed
object. Subclasses can extend the table with their own internal
commands.

**Four built-in internal commands:**

- **`-dump-spec`** - introspects a running modulino and writes a YAML
  manifest to the project root. Accepts an optional `roles` argument
  to map commands to role class names instead of sub names
- **`-scaffold`** - generates a role-based project tarball containing
  role stubs, a slimmed main module with extracted POD, a `project.mk`
  with inter-module dependencies, and the YAML manifest. Accepts an
  optional spec file path for scaffolding from an existing manifest
- **`-migrate`** - runs `-dump-spec roles` then `-scaffold` in one
  step
- **`-generate-completion`** - generates a bash completion script from
  the live command list and option specs. Requires `MODULINO_WRAPPER`
  to be set in the environment

**`cli-simple` bash wrapper.** A new executable allowing new projects
to be scaffolded without an existing CLI::Simple script:

```bash
cli-simple -scaffold my-project.yml
```

## Changes

**`choose()` now respects wantarray context** - previously always
returned a scalar. This is a behavior change for callers relying on
list context; review any existing uses of `choose`.

**`main()` no longer croaks without a manifest** - falls back to a
minimal `{ default => \&usage }` command table, allowing `CLI::Simple`
itself to be invoked as a modulino.

**`set_args()` added** - resets positional arguments on the current
object.

**License changed from MIT to Perl artistic license** - aligns with
the broader CPAN::Maker ecosystem.

## New Dependencies

`Role::Tiny`, `Role::Tiny::With`, `YAML::XS`, `Archive::Tar`, `File::ShareDir`, `File::Which`

## New Modules

`CLI::Simple::Helpers`, `CLI::Simple::DumpSpec`, `CLI::Simple::Migrate`, `CLI::Simple::Scaffold`

## See Also

`CPAN::Maker::Bootstrapper` 2.0.0 provides the `import-scaffold`
command to consume the tarball generated by `-scaffold` and produce a
complete buildable CPAN distribution in one step.
